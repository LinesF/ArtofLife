# 📖 Art of Life - 개인 홈페이지 운영 및 사용 설명서

본 저장소는 HTML, CSS, JavaScript로 이루어진 정적인 개인 홈페이지입니다. 마크다운(`.md`) 기반의 글 작성 및 정적 빌더(`build.py`)가 탑재되어 있어, 간편하게 홈페이지를 운영하고 깃허브 Pages에 자동으로 배포할 수 있습니다.

---

## 🚀 1. 글을 새로 올릴 때 (Create)

새로운 게시글을 홈페이지에 업로드하는 방법입니다.

1. **마크다운 파일 생성**:
   `_posts/` 디렉토리에 새로운 마크다운 파일(예: `2026-07-12-my-new-post.md`)을 생성합니다.
   * 파일명은 한글이나 영어 모두 가능하나, 직관적인 관리를 위해 `YYYY-MM-DD-제목.md` 형식을 권장합니다.

2. **머리말(Frontmatter) 입력**:
   파일의 맨 첫 줄에 반드시 다음 형식으로 글의 메타데이터를 선언하고, 그 아래에 본문을 적어줍니다:
   ```markdown
   ---
   title: 올릴 글의 제목을 적어주세요
   category: 일기장  # [일기장, Notes, Projects, 사색 일지] 중 하나를 적어주세요.
   date: 2026-07-12  # YYYY-MM-DD 형식으로 적어주세요.
   ---
   여기에 본문 내용을 마크다운으로 자유롭게 작성합니다.
   
   ## 본문 부제목
   본문 텍스트가 시작됩니다. 마크다운의 굵게(**텍스트**), 리스트(- 아이템), 인용구(> 문장) 등이 지원됩니다.
   ```

3. **분류(Category) 선택 기준**:
   * `일기장`: 소소한 일상, 가벼운 잡담, 당일의 일기 기록
   * `Notes`: 공부한 기술 개념, 정리한 전공 지식, 아카이빙할 지식 조각
   * `Projects`: 개인 연구, 포트폴리오 성과물, 프로젝트 기록
   * `사색 일지`: 깊은 생각, 문득 떠오른 단상이나 수필

4. **갱신(빌드)하기**:
   두 가지 방법 중 하나를 선택합니다:
   * **로컬 빌드**: 터미널에서 `python build.py`를 실행하여 로컬 리소스를 갱신합니다.
   * **원격 자동 빌드**: 깃허브 저장소에 변경 사항을 `git push`하면, GitHub Actions 서버가 알아서 빌드를 수행해 배포합니다.

## ✏️ 2. 올린 글을 수정할 때 (Update)

이미 작성하여 올라간 글의 내용이나 제목을 고치는 방법입니다.

1. **마크다운 파일 수정**:
   `_posts/` 디렉토리에 있는 해당 마크다운 원본 파일을 엽니다.
   * **본문 수정**: 오타를 고치거나 문장을 가다듬습니다.
   * **메타데이터 수정**: 제목(`title`)이나 날짜(`date`), 혹은 분류(`category`)를 자유롭게 바꿉니다.

2. **갱신(빌드)하기**:
   수정이 완료되면, 다시 빌드 프로세스(`python build.py` 실행 또는 `git push`)를 통해 사이트를 갱신합니다.
   * **참고**: 빌더가 기존에 컴파일되어 있던 낡은 HTML 파일을 덮어쓰고, 목록 페이지와 검색 데이터베이스까지 실시간으로 알아서 재조립하여 동기화해 줍니다.

---

## 🗑️ 3. 글을 완전히 삭제할 때 (Delete)

홈페이지에서 특정 글을 걷어내고 싶을 때 수행하는 방법입니다.

1. **마크다운 파일 삭제**:
   `_posts/` 디렉토리에 있는 해당 마크다운 파일(`.md`)을 지웁니다.

2. **기존 생성된 HTML 파일 수동 정리**:
   해당 글이 들어가 있었던 카테고리 폴더(예: `diary/`)로 이동하여, 컴파일되어 존재하던 해당 HTML 파일(예: `2026-07-12-my-new-post.html`)을 수동으로 삭제해 줍니다.

3. **목록 및 검색 데이터 리빌드**:
   터미널에서 `python build.py`를 실행하거나 깃허브에 변경을 푸시합니다.
   * 빌더가 돌면서 지워진 글을 목록(index.html) 테이블에서 제외하고, `search.html` 내의 검색 데이터베이스에서도 흔적을 깔끔하게 지워 줍니다.

## 🖥️ 4. 로컬 빌더 명령어 가이드 (`build.py`)

로컬 컴퓨터에서 오프라인 상태로 동작 및 빌드를 미리 테스트해 보고 싶을 때 유용한 명령어입니다.

* **빌드 수행하기**:
  프로젝트 루트 디렉토리에서 터미널을 열고 아래 명령어를 입력합니다.
  ```bash
  python build.py
  ```
  실행 결과로 개별 HTML이 컴파일되며, 목록 동기화가 성공적으로 끝났다는 로그가 출력됩니다.

---

## 🚀 5. 깃허브 push 및 자동 라이브 배포

홈페이지 코드를 깃허브 저장소([https://github.com/LinesF/ArtofLife](https://github.com/LinesF/ArtofLife))로 동기화하고 웹상에 라이브 배포하는 최종 단계입니다.

1. **Git을 통한 업로드**:
   글을 추가, 수정 혹은 삭제한 뒤 아래 명령어를 차례로 입력하여 깃허브에 푸시합니다:
   ```bash
   git add .
   git commit -m "docs: write new post YYYY-MM-DD"
   git push origin main
   ```

2. **자동 배포 파이프라인 (GitHub Actions)**:
   * 푸시하는 즉시 깃허브 서버 내의 워크플로우(`.github/workflows/deploy.yml`)가 자동으로 가동됩니다.
   * 서버가 `build.py`를 원격 구동하여 완벽한 상태의 정적 홈페이지를 빌드하고, 자동으로 최종 깃허브 Pages 링크에 퍼블리싱을 완료해 줍니다.
   * 진행 상황은 깃허브 저장소 페이지의 **Actions** 탭에서 한눈에 모니터링할 수 있습니다.
